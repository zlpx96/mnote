# mnote 产品功能文档

mnote 是一个移动端 PWA 应用，通过 GitHub API 浏览私有仓库中的 Markdown 文件。无需服务器，所有数据存储在本地设备，部署在 GitHub Pages。

---

## 核心功能

### 1. Token 认证

- 首次打开进入 Setup 页面，输入 GitHub Personal Access Token
- Token 需要 `repo` 权限（支持读取私有仓库）
- 验证通过后 Token 存入本地 localStorage，之后无需重复输入
- Token 过期或失效时，任意 API 请求会自动跳转回 Setup 页面重新登录
- 支持手动重置 Token（仓库列表页右上角 ⚙ 按钮）
- 重置 Token 时自动清除所有本地缓存和滚动记录

### 2. 仓库管理

- 支持添加多个仓库，仓库列表以卡片形式展示
- 搜索添加仓库：支持搜索自己名下及加入的组织的所有有权限的仓库
- 支持删除已添加的仓库
- 仓库列表持久化存储在 localStorage

### 3. 文件树浏览

- 进入仓库后展示文件树，目录排在文件前面
- 支持点击目录展开子目录，无限层级
- 顶部 breadcrumb 路径导航，可点击任意层级快速跳转
- 只有 `.md` 文件可以点击打开，其他文件类型置灰显示
- 加载失败时显示错误信息，API 限流时显示重置时间

### 4. Markdown 阅读

- 支持基础 Markdown 渲染：标题、加粗、斜体、列表、代码块、引用、分割线、链接
- 字体大小 16px，行高 1.7，适合手机阅读
- 正确支持中文（UTF-8 解码）
- 禁止渲染原始 HTML 标签（防 XSS）

### 5. 离线缓存

- 读取过的 MD 文件内容自动缓存到 localStorage
- 断网时自动使用缓存内容，静默失败不报错
- 有网络时每次打开文件都会后台刷新缓存

### 6. 阅读进度记录

- 自动记录每篇文章的滚动位置
- 再次打开同一文件时自动恢复到上次读到的位置

### 7. 收藏

- 阅读页右上角 ☆/★ 按钮，点击收藏或取消收藏
- 仓库列表页点击 ★ 按钮展开收藏面板
- 收藏面板显示所有收藏文章，点击直接跳转阅读
- 收藏数据持久化存储在 localStorage

### 8. 导航

- 返回按钮优先返回上一个 mnote 内的页面
- 若上一页不在 mnote 内（如从收藏直接打开），返回到仓库列表页

### 9. PWA 支持

- 支持"添加到主屏幕"，以独立应用模式运行（无浏览器导航栏）
- Service Worker 缓存应用资源，支持离线启动
- 主题色 #0969da

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| Token 无效或过期 | 自动跳转到 Setup 页面重新登录 |
| API 限流（5000次/小时） | 显示限流提示及重置时间 |
| 网络断开 | 显示缓存内容，静默失败 |
| 网络断开且无缓存 | 显示加载失败错误信息 |

---

## 数据存储

所有数据存储在用户设备的 localStorage，不经过任何服务器：

| 数据 | Key 前缀 |
|------|----------|
| GitHub Token | `mnote_token` |
| 仓库列表 | `mnote_repos` |
| 文件内容缓存 | `mnote_cache_` |
| 滚动位置 | `mnote_scroll_` |
| 收藏列表 | `mnote_favorites` |

## 技术栈

- **框架：** Vue 3 + Vite
- **路由：** Vue Router 4（Hash History）
- **Markdown 渲染：** marked.js
- **部署：** GitHub Pages（GitHub Actions 自动部署）
- **认证：** GitHub Personal Access Token（仅存本地）
